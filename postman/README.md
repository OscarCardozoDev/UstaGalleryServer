# UstaGallery API — Postman Test Suite

Newman integration tests for the NestJS backend. Tests run sequentially; auth cookies are shared across requests within a run.

## Setup

```bash
# from server/
bun run test:api       # runs Newman against localhost:3000; report → reports/
```

Requires:
- Server running on port 3000 (`bun run start:dev`)
- DB seeded (`bun run prisma:seed:static`)
- `server/postman/environments/testing.env.json` values match your local env

## File structure

```
postman/
├── build-collection.js          # Source of truth — edit here, then rebuild
├── collections/server-api/
│   └── collection.json          # Generated by build-collection.js (do not edit directly)
├── environments/
│   └── testing.env.json         # Env vars (baseUrl, credentials, dynamic IDs)
├── POSTMAN_PROGRESS.md          # Per-endpoint test coverage tracker
└── README.md                    # This file
```

To rebuild after editing `build-collection.js`:

```bash
node postman/build-collection.js
```

## Modules

### Auth
Handles registration, login, logout, and email verification. Uses **HttpOnly cookies** (`access_token`) — no `Authorization` header needed after login.

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | /auth/register | — | Create credentials, sets cookie |
| POST | /auth/login | — | Validate credentials, sets cookie |
| POST | /auth/logout | — | Clears cookie |
| POST | /auth/send-code | cookie | Send 6-digit verification code by email (Resend) |
| POST | /auth/verify-code | cookie | Validate the 6-digit code, marks email verified |
| GET | /auth/without-profile | cookie + admin | List credentials with no linked user profile |

### User
Profile management. A user exists in two tables: `Credentials` (auth) and `Users` (profile). `hasProfile: false` means credentials exist but no profile row yet.

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | /user/create | cookie | Create user profile (any role, uses JWT uid) |
| GET | /user/allActive | cookie + admin | List all active user profiles |
| GET | /user/me | cookie | Get own profile |
| GET | /user/author/:uid | — | Public author info (name, bio, photo) |
| GET | /user/:uid | — | Full user profile by uid |
| PUT | /user/update | cookie | Update own profile |
| PUT | /user/update-photo | cookie | Replace own profile photo |
| PUT | /user/update-photo/:uid | cookie + admin | Replace profile photo for any user |
| PUT | /user/:uid | cookie + admin | Update any user's profile |
| PATCH | /user/desactivate | cookie | Deactivate own account |
| PATCH | /user/:uid/desactivate | cookie + admin | Deactivate any user |
| PATCH | /user/:uid/reactivate | cookie + admin | Reactivate a deactivated user |

### Groups
Academic groups linking students to a professor. Students can belong to multiple groups.

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | /groups/create | cookie + admin | Create group with professor and optional student list |
| GET | /groups/get | cookie | Paginated list of groups |
| GET | /groups/get/:uid | cookie | Group by uid |
| PUT | /groups/update/:uid | cookie + admin | Update group name |
| PATCH | /groups/change-profesor/:uid | cookie + admin | Assign new professor to group |
| POST | /groups/student/add | cookie + admin | Add student(s) to group(s) |
| GET | /groups/student/get/:uid | cookie | List students in a group |
| PUT | /groups/student/update/:uid | cookie + admin | Replace full student list in group |
| DELETE | /groups/student/delete/:uid | cookie + admin | Remove one student from group |
| DELETE | /groups/student/deleteAll/:uid | cookie + admin | Remove all students from group |
| DELETE | /groups/delete/:uid | cookie + admin | Delete group |

### Photos
Base64 image storage. Photos are saved to DB and written to `public/images/` as files. Max body size: 6 MB.

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | /photos/create | cookie | Upload photo (base64, name, folder) |
| GET | /photos/get/:uid | cookie | Get photo by uid |
| PUT | /photos/edit/:uid | cookie | Replace photo base64 |

### Products
Artworks submitted by students. Status flow: `PENDING → APPROVED/REJECTED`. Products link to authors, styles, and photos.

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | /products/create | cookie + professor | Create product with authors, styles, images |
| GET | /products/getGalleryHome | — | Paginated approved products for public gallery |
| GET | /products/getAuthor/:uid | — | Paginated products by author |
| GET | /products/get/:uid | — | Product detail by uid |
| GET | /products/getGroup/:uid | cookie + professor/admin | Products by group |
| GET | /products/getAll | cookie + admin | All products (any status) |
| PUT | /products/update/:uid | cookie + professor | Update product (resets to PENDING) |
| PATCH | /products/status/:uid | cookie + professor | Update status (REJECTED requires feedback) |
| PUT | /products/approveMany | cookie + professor | Bulk approve products |

### Styles
Art style tags associated to a group. Students use styles when creating products.

| Method | Path | Auth | Description |
|---|---|---|---|
| GET | /styles/all | — | All styles |
| GET | /styles/all/:groupId | — | Styles by group |
| GET | /styles/get/:uid | — | Style by uid |
| POST | /styles/create | cookie + professor | Create style for a group |
| PUT | /styles/update/:uid | cookie + professor | Update style name/description |
| DELETE | /styles/delete/:uid | cookie + admin | Delete style |

### Events
Exhibition events. Status flow: `PENDING → APPROVED → COMPLETED` (lazy) or `CANCELLED`. Groups can be invited; coordinator's groups link directly.

| Method | Path | Auth | Description |
|---|---|---|---|
| GET | /events/upcoming | — | Paginated upcoming events |
| GET | /events/past | — | Paginated past events |
| GET | /events/home | — | Featured events for home page |
| GET | /events/getByGroup/:uid | — | Events linked to a group |
| GET | /events/get/:uid | — | Event detail |
| GET | /events/getAll | cookie + admin | All events (any status) |
| GET | /events/available-products/:groupId | cookie + professor | Products eligible to add to event |
| GET | /events/invitations/pending | cookie + professor | Pending invitations for professor's groups |
| POST | /events/create | cookie + professor | Create event |
| PUT | /events/update/:uid | cookie + professor | Update event (resets to PENDING) |
| PUT | /events/:uid/products | cookie + professor | Replace product list on event |
| PATCH | /events/status/:uid | cookie + admin | Change event status (CANCELLED requires feedback) |
| POST | /events/:uid/invite | cookie + admin | Invite a group to event |
| DELETE | /events/:uid/invite/:groupId | cookie + admin | Revoke group invitation |
| PATCH | /events/deactivate/:uid | cookie + admin | Deactivate event |

### Classes
Individual class sessions within a group. Can be auto-generated from a Schedule or created manually.

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | /classes/create | cookie + professor | Create manual class session |
| GET | /classes/group/:uid | cookie + professor/admin | Classes by group (optional date range filter) |
| GET | /classes/current/:groupId | cookie | Active class for group right now |
| PATCH | /classes/:uid/topic | cookie + professor | Set/update topic and review for a class |
| GET | /classes/:uid/attendance | cookie + professor/admin | Attendance list for a class |
| POST | /classes/attend | cookie + student | Mark own attendance for a class |

### Schedule
Recurring weekly slots. Creating a schedule auto-generates Class rows from the next occurrence until `SEMESTER_END_DATE`. Editing/deleting a schedule regenerates future unattended classes.

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | /schedule/create | cookie + professor | Create weekly recurring slot |
| GET | /schedule/group/:uid | cookie + professor/admin | Schedules for a group |
| PUT | /schedule/:uid | cookie + professor | Update slot (regenerates future classes) |
| DELETE | /schedule/:uid | cookie + professor | Delete slot (removes future unattended classes) |
